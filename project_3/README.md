# Лабораторна робота №3: Кастомні жести та стилізація
## 🎯 Мета
Навчитися працювати з жестами користувача (свайпи, тапи) та застосувати сучасні підходи до стилізації.
## 🛠 Що було використано
- **Жести:** Бібліотека `react-native-gesture-handler`.
- **Стан:** `Context API` для збереження прогресу гри та теми оформлення.
- **Стилізація:** `styled-components` для React Native (світла/темна теми).
## 📱 Опис застосунку
Розроблено гру-клікер. Користувач взаємодіє з екраном за допомогою жестів (наприклад, подвійний тап або довге натискання), щоб заробляти "монетки". Інтерфейс повністю стилізовано через Styled Components, а глобальний стейт зберігає налаштування поточної теми (Світла/Темна), яка миттєво змінює вигляд всього додатку.

## ⚙️ Як це працює
Архітектура застосунку побудована на взаємодії трьох основних блоків: глобального стану, системи розпізнавання жестів та динамічної типізації стилів.
1. **Глобальний стан (Context API):** 
   У корені застосунку створено `Provider`, який зберігає два ключових параметри: кількість зароблених "монеток" та поточну тему (світла/темна). Завдяки цьому будь-який компонент (чи то екран гри, чи екран налаштувань) має миттєвий доступ до цих даних без необхідності передавати пропси (prop drilling).
2. **Обробка жестів (Gesture Handler):**
   Замість стандартних кнопок (`TouchableOpacity` чи `Pressable`), ігрова зона використовує `react-native-gesture-handler`. За допомогою `GestureDetector` налаштовано відслідковування специфічних жестів:
   - **Double Tap (Подвійний тап):** наприклад, додає бонусні монетки або змінює тему.
   - **Long Press (Довге натискання):** активує режим автоматичного збору або скидає прогрес.
   Кожен успішний жест викликає функцію оновлення стану в Context API.
3. **Динамічна стилізація (Styled Components):**
   Додаток використовує `ThemeProvider` від `styled-components`. Коли в глобальному стані змінюється тема, `ThemeProvider` отримує новий об'єкт з кольорами (палітру). Всі UI-компоненти (контейнери, тексти, кнопки), створені через `styled`, автоматично і плавно перемальовуються з урахуванням нових кольорів без перезавантаження застосунку.
4. **Ігрова механіка:**
   Користувач взаємодіє з екраном за допомогою жестів $\rightarrow$ `Gesture Handler` фіксує дію $\rightarrow$ Оновлюється лічильник в `Context API` $\rightarrow$ React миттєво оновлює UI, відображаючи новий баланс гравця.
