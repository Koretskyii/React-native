# Лабораторна робота №2: Вкладена навігація та списки

## 🎯 Мета
Опанувати роботу зі складними навігаційними structures та ефективне відображення великих масивів даних.

## 🛠 Що було використано
- **Навігація:** Вкладені навігатори (Drawer Navigator містить у собі Stack Navigator).
- **Списки:** Оптимізовані компоненти `FlatList` та `SectionList`.

## 📱 Опис застосунку
Додаток має бічне меню (Drawer), через яке можна перейти до різних розділів. Основний функціонал зосереджений на екранах зі списками: реалізовано звичайний список новин (`FlatList`) та згрупований список контактів (`SectionList`). Додано можливість "потягнути вниз" для оновлення даних.

## Як це працює
Ієрархія навігації (Вкладеність)

Кореневим навігатором виступає Drawer Navigator, який створює глобальне бічне меню.
Кожен окремий розділ у меню («Новини», «Контакти») містить у собі власний Stack Navigator. Це дозволяє користувачу переходити вглиб розділу (наприклад, від загального списку до деталей конкретної новини), зберігаючи при цьому можливість у будь-який момент відкрити бічне меню.
Екран «Новини» (FlatList)

Дані масиву новин відображаються за допомогою компонента FlatList. На відміну від звичайного ScrollView, він рендерить лише ті елементи, які зараз знаходяться на екрані, що забезпечує високу продуктивність.
Вбудовано підтримку жесту Pull-to-Refresh (потягнути для оновлення) за допомогою пропсів refreshing та onRefresh.
При натисканні на картку новини відбувається перехід на сторінку деталей завдяки динамічному роутингу Expo Router (через файли типу [id].tsx).
Екран «Контакти» (SectionList)

Сирі дані контактів програмно обробляються: сортуються за алфавітом і розбиваються на об'єкти-секції за першою літерою імені.
Оброблений масив передається у компонент SectionList, який виводить контакти групами. Заголовки секцій (літери) закріплюються при скролінгу, створюючи інтерфейс класичної телефонної книги.
