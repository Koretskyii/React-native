# Лабораторна робота №4: Робота з файловою системою
## 🎯 Мета
Опанувати механізми роботи з локальною пам'яттю мобільного пристрою.
## 🛠 Що було використано
- **File System:** Сучасне об'єктно-орієнтоване API `expo-file-system` (`Directory`, `File`).
- **UI Компоненти:** Кастомні модальні вікна (`Modal`), Alert для підтвердження.
## 📱 Опис застосунку
Створено повноцінний мобільний "Файловий менеджер". Застосунок дозволяє переглядати вміст папок, створювати нові текстові файли та директорії, читати вміст файлів та безпечно їх видаляти. Реалізована навігація для повернення на рівень вище.

## ⚙️ Як це працює
1. **Доступ до файлової системи:** Додаток використовує `documentDirectory` пристрою як базову (кореневу) директорію для зберігання даних. За допомогою об'єктно-орієнтованого підходу `expo-file-system` створюється початковий екземпляр `Directory`.
2. **Навігація папками:** При відкритті папки додаток зчитує її вміст та оновлює стан поточного шляху. Реалізовано зручний механізм навігації — додано спеціальний елемент для повернення "на рівень вище" (до батьківської директорії).
3. **Створення елементів:** Для створення нових текстових файлів або директорій викликаються кастомні компоненти `Modal`. Після вводу назви користувачем, застосовуються відповідні методи API для створення об'єктів у поточній директорії.
4. **Взаємодія з файлами:** 
   - **Читання:** При натисканні на існуючий файл викликається метод читання його вмісту, який відображається на екрані.
   - **Видалення:** Для безпеки користувача видалення файлу чи папки супроводжується запобіжним `Alert`. Після підтвердження елемент видаляється з пам'яті.
5. **Синхронізація UI:** Усі операції (створення, видалення, перехід між папками) пов'язані з React State. Це дозволяє миттєво оновлювати список файлів на екрані після кожної дії, забезпечуючи актуальність відображених даних.
