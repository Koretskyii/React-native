# Лабораторна робота №6: Firebase Auth та Firestore
## 🎯 Мета
Інтеграція хмарних сервісів Google Firebase для повноцінної автентифікації та збереження даних у віддаленій базі.
## 🛠 Що було використано
- **BaaS:** Firebase (`firebase/auth`, `firebase/firestore`).
- **Функціонал:** `signInWithEmailAndPassword`, `updateDoc`, `reauthenticateWithCredential`.
## 📱 Опис застосунку
Фінальний додаток містить реальну систему акаунтів. Користувач може зареєструватися, скинути пароль або зайти у свій профіль. У профілі можна вказати своє ім'я, вік та місто — ці дані зберігаються у Firestore. У базі налаштовані Security Rules, що забороняють користувачам читати чужі дані.

⚙️ **Як це працює**
- **Автентифікація:** Реалізована за допомогою модуля `firebase/auth`. Додаток підтримує повний цикл керування акаунтом: реєстрацію, вхід (`signInWithEmailAndPassword`) та скидання пароля. Для чутливих операцій (наприклад, зміни облікових даних) передбачена повторна автентифікація користувача (`reauthenticateWithCredential`).
- **Хмарна база даних:** Персональні дані користувача (ім'я, вік, місто) зберігаються у NoSQL базі даних Firestore. Для оновлення інформації в документах користувачів використовується метод `updateDoc`.
- **Правила безпеки (Security Rules):** На стороні сервера Firebase налаштовані суворі правила доступу. Вони гарантують, що читати або редагувати документ у колекції користувачів може лише той користувач, чий `uid` збігається з ідентифікатором документа. Це унеможливлює витік даних.
- **Захищений роутинг:** Додаток відслідковує активну сесію Firebase. Якщо користувач не авторизований, він має доступ лише до екранів входу/реєстрації; після успішного входу він автоматично отримує доступ до захищеного екрану профілю.
